# etcd分布式存储

## 简介
etcd是一个高可用的分布式**键值**数据库。内部采用raft协议作为一致性算法，etcd基于go语言实现，提供RESTful风格访问接口操作方便。

## 启动服务器
在命令行终端输入：`etcd`

## etcd常用命令
使用`etcd --help`命令可以查看etcd的所有命令。

- **put**
  - `etcdctl --endpoints=$ENDPOINTS put foo "Hello World!"`
    - `--endpoints=$ENDPOINTS`是服务器的端点IP，如果不写默认是本地。
- **get**
  - `etcdctl --endpoints=$ENDPOINTS get foo`
  - `etcdctl --endpoints=$ENDPOINTS --write-out="json" get foo`
    - `--write-out="json"`：输出json格式
- **del**
  - `etcdctl --endpoints=$ENDPOINTS del key`
- **watch（监视变化）**
  - `etcd --endpoints=$ENDPOINTS watch stock1`

## golang操作etcd
```go
package main

import (
	"context"
	"fmt"
	"time"

	"go.etcd.io/etcd/client/v3"
)

func main() {
	var (
		config  clientv3.Config
		client  *clientv3.Client
		err     error
		kv      clientv3.KV
		getResp *clientv3.GetResponse
	)

	config = clientv3.Config{
		Endpoints:   []string{"127.0.0.1:2379"},
		DialTimeout: 5 * time.Second,
	}

	// 建立一个客户端
	if client, err = clientv3.New(config); err != nil {
		fmt.Println(err)
		return
	}
    defer client.Close()

	// 用于读写etcd的键值对
	kv = clientv3.NewKV(client)

	// 写入
	kv.Put(context.TODO(), "site", "golang-tech-stack")
	kv.Put(context.TODO(), "name", "老郭")

	// 读取name为前缀的所有key
	if getResp, err = kv.Get(context.TODO(), "site", clientv3.WithPrefix()); err != nil {
		fmt.Println(err)
		return
	} else {
		// 获取成功
		fmt.Println(getResp.Kvs)
	}

	// 删除name为前缀的所有key
	if _, err = kv.Delete(context.TODO(), "name", clientv3.WithPrevKV()); err != nil {
		fmt.Println(err)
		return
	}
}
```

## etcd架构解析
etcd的核心价值在于提供了可靠的分布式数据存储，其强一致性和高可用性是通过Raft算法实现的，而MVCC机制提供了高效的并发控制和历史版本管理。
