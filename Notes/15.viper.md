# viper

viper是适用于go应用程序的完整配置解决方案，它被设计用于在应用程序中工作，并且可以处理所有类型的配置需求和格式。支持以下特性：
- 设置默认值
- 从json、toml、yaml、envfile和java properties格式的配置文件读取配置信息
- 实时监控和重新读取配置文件
- 从环境变量中读取
- 从远程配置系统（etcd或consul）读取并监控配置变化
- 从命令行参数读取配置
- 从buffer读取配置
- 显示配置值

## 核心功能
1. 支持多种配置格式
```
viper.SetConfigType("yaml")
viper.SetConfigFile("config.yaml")
```

2. 加载配置文件
```
viper.SetConfigName("config")   // 不带扩展名
viper.AddConfigPath(".")   // 设置配置文件路径
err := viper.ReadInConfig()   // 读取配置文件
```

3. 访问配置值: 通过键来访问配置值，支持多种数据类型
```
name := viper.GetString("name")
age := viper.GetInt("age")
married := viper.GetBool("married")
```

4. 环境变量支持
```
viper.BindEnv("name")

// 可以设置环境变量前缀，使得所有相关的环境变量都具有统一的前缀
viper.SetEnvPrefix("APP")
viper.BindEnv("name")
```

5. 处理命令行标志：viper可以与Pflag或flag结合使用，从命令行参数中读取配置
```
pflag.String("name", "Anonymous", "Your name")
pflag.Parse()
viper.BindFlags(pflag.COmmandLine)
```
viper将会自动绑定pflag的命令行标志，并将其作为配置的一部分

6. 支持远程配置
```
viper.AddRemoteProvider("consul", "localhost:8500", "config")
viper.SetConfigType("json") // 因为 Consul 中存储的是 json
viper.ReadRemoteConfig()
```

7. 动态配置更新：viper可以监听配置文件或远程配置的变化，并自动更新配置值。
```
viper.WatchConfig()
viper.OnConfigChange(func(e fsnotif.Event) {
    fmt.Println("Config file changed:", e.Name)
})
```
