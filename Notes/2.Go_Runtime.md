# Runtime 

## Goroutine原理
Goroutine是一个与其他goroutines并行运行在同一地址空间的Go函数或方法。一个运行的程序由一个或多个goroutine组成。

Goroutines在同一个用户地址空间里并行独立执行functions，channels则用于goroutines间的通信和同步访问控制。

### goroutine和thread的区别
1. Thread开销很大。创建线程需要调用内核，分配内核栈（固定大小，1MB-8MB），并注册到调度器。销毁时还需要回收资源。Goroutine开销小。创建只需要在用户态分配初始栈（2KB），虽需增长。
2. 调度方式不同。Thread是抢占式，由操作系统内核调度器决定哪个线程运行、运行多久、何时切换。切换涉及用户态到内核态的上下文切换，成本高；Goroutine是协作式+抢占式。由go运行时自己的调度器（GMP模型）管理。切换点发生在函数调用、通道操作、系统调用等时机。
3. Thread切换速度慢，需要保存恢复cpu寄存器、程序计数器、内存映射表等，并陷入内核。Goroutine速度快，完全在用户态，仅需保存少量寄存器，上下文切换成本低于线程。
4. Thread可以天然被os调度到不同的cpu核心上并行执行；goroutine逻辑上并发，物理上通过GOMAXPROCS设置P的数量，从而将goroutine分配给若干os线程实现并行。一个阻塞的goroutine不会阻塞其他goroutine,因为运行时会把当前线程上的其它goroutine迁移到别的线程。
