# zap

## zap两种创建方式
```go
// 1. 开发模式，适合本地开发，彩色、易读、带行号
logger, _ := zap.NewDevelopment()

// 2. 生产模式（JSON格式），适合线上，结构化、可被日志平台解析
logger, _ := zap.NewProduction()

defer logger.Sync()  // 刷新缓冲区
```

## 日志级别
```
Debug → 调试
Info  → 普通信息
Warn  → 警告
Error → 错误
Panic → 崩溃（打日志后 panic）
Fatal → 致命错误（打日志后 os.Exit(1)）
```

## sugar
zap提供了两种类型的日志记录器，Sugared logger 和 logger。

sugar 没有 logger快，但是比它更方便。
```go
logger, _ := zap.NewDevelopment()
sugar := logger.sugar()

sugar.Infow()
sugar.Infof()
```

## zap的文件输出
```go
func NewLogger() (*zap.Logger, error) {
    cfg := zap.NewProductionConfig()
    cfg,OutputPaths = []string {
        "./myproject.log",
    }
    return cfg.Build()
}
```

## 简写
```go
logger, _ := zap.NewDevelopment()
zap.ReplaceGlobals(logger)

zap.S().Infof()  // zap.L()
```

`zap.ReplaceGlobals(logger)`会把上面创建的logger设置成全局logger,整个项目的任何地方都能直接用zap日志，不会把logger到处传递。
