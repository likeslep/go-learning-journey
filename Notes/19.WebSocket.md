# WebSocket

WebSocket，基于TCP的应用层协议，通过HTTP升级机制建立连接，切换到WebSocket协议进行全双工通信。

客户端发送 `HTTP Upgrade Request`，服务端响应 `HTTP 101 Switching Protocols`之后，由HTTP协议转换为WebSocket协议。WebSocket以帧为单位传输数据，帧包括头和负载数据。WebSocket的心跳机制，通过发送Ping帧和Pong帧来保持连接活跃。

## WebSocket与HTTP对比
|特性|HTTP|WebSocket|
|---|---|---|
|连接方式|短连接/长连接|持久连接|
|通信模式|请求-响应|全双工|
|服务器推送|不支持（需轮询）|原生支持|
|延迟|10-300ms|<1ms（已连接）|
|头部开销|200-800字节|2-14字节|
|适用场景|RESTful API、静态资源|实时通信、高频推送|
|电池消耗|高（频繁请求）|低（长连接）|


## 实例：IM即时通讯中WebSocket的使用

这里展示的是我在做IM即时通讯程序中，实现用户与用户的通信阶段时的代码。在此之前，我已经实现了用户模块的注册和登录功能。当用户登录成功后，会得到他的token，用户在连接WebSocket时，需要带上他们的token，以验证当前用户为登录状态，因为只有用户在登录后才能发送消息。

**WebSocket连接建立层代码（核心部分）：**
```go
package api

import (
	"net/http"
	"server/internal/chat"
	"server/pkg"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/gorilla/websocket"
	"go.uber.org/zap"
)
// 升级为 WebSocket
var upGrader = websocket.Upgrader{
	ReadBufferSize:  1024,
	WriteBufferSize: 1024,
	CheckOrigin: func(r *http.Request) bool {
		return true // 允许跨域
	},
}

// WebSocketConnect 建立连接
func WebSocketConnect(c *gin.Context) {
	// 1. 获取 Token
	token := c.Query("token")
	if token == "" {
		c.JSON(401, gin.H{"msg": "缺少token"})
		return
	}

	// 2. 解析 Token
	claims, err := pkg.ParseToken(token)
	if err != nil {
		c.JSON(401, gin.H{"msg": "token无效"})
		return
	}
	userID := claims.UserID

	// 3. 升级协议
	conn, err := upGrader.Upgrade(c.Writer, c.Request, nil)
	if err != nil {
		zap.S().Errorf("升级ws失败: %v", err)
		return
	}

	// 4. 创建客户端
	client := &chat.Client{
		UserID:    userID,
		Conn:      conn,
		Send:      make(chan []byte, 256),
		Heartbeat: time.Now(),
	}

	// 5. 注册到管理器
	chat.H.Register(userID, client)

	// 6. 启动读写协程
	go client.Write()
	go client.Read()
}
```
项目中，我使用的gin框架来建立web，并且把WebSocket集成到gin中。`WebSocketConnect` 函数为WebSocket连接处理函数。联合我配置好的路由：`public.GET("/ws", api.WebSocketConnect)`，当前端调用对应的url时，就会调用WebSocketConnect函数建立连接。请求建立连接时，需要在url中加入参数token的值，函数中对提供的token值进行解析来识别身份，然后将当前的协议由HTTP升级为WebSocket，给当前用户创建好一个客户端并将其注册到**连接管理中心（hub.go）**（连接管理中心会记录当前在线的所有用户，对所有连接用户进行集中管理。）然后开启两个协程Write()和Read()，Read()的工作是监听客户端给服务端发来的消息，当客户端发来消息时，服务端对消息进行识别以判断消息的接收者是谁，通过接受者的Write()协程来进行消息转发；Write()的工作是服务端给客户端发送消息，当客户端在连接过程中没有其他用户给它发送消息时，服务端会自动向客户端发送Ping帧，客户端收到服务端的Ping帧后，会向服务端返回Pong帧来刷新心跳来维持连接的活跃。

**消息结构体：**
```go
// Message 客户端与服务端通信的消息结构体
type Message struct {
	Type     int    `json:"type"`      // 消息类型 1单聊 2群聊
	ToUserId int64  `json:"to_user_id"`// 接收者ID
	Content  string `json:"content"`   // 消息内容
	MsgType  int    `json:"msg_type"`  // 1文本
}
```

**连接管理中心(hub.go)：**
```go
package chat 


import (
	"go.uber.org/zap"
	"sync"
)

type Hub struct {
	// 在线用户列表：key=userID, value=*Client
	OnlineUsers map[int64]*Client 
	mutex sync.RWMutex 
}

// 全局单例
var H = &Hub {
	OnlineUsers: make(map[int64]*Client),
}

// 注册连接（用户上线）
func (h *Hub) Register(userID int64, c *Client) {
	h.mutex.Lock()
	defer h.mutex.Unlock()
	h.OnlineUsers[userID] = c
	zap.S().Infof("用户[%d]上线，当前在线：%d", userID, len(h.OnlineUsers))
}

// 注销连接（用户下线）
func (h *Hub) UnRegister(userID int64) {
	h.mutex.Lock()
	defer h.mutex.Unlock()
	delete(h.OnlineUsers, userID)
	zap.S().Infof("用户[%d]下线，当前在线：%d", userID, len(h.OnlineUsers))
}

// 获取用户在线状态
func (h *Hub) GetClient(userID int64) (*Client, bool) {
	h.mutex.RLock()
	defer h.mutex.RUnlock()
	client, ok := h.OnlineUsers[userID]
	return client, ok 
}
```
这里展示了连接管理中心是如何实现的，其实就是定义了一个枢纽结构体Hub，其中包括了存放用户列表的map和读写互斥锁。接着我们对这个结构体实现了一个全局单例，整个程序只能有一个连接管理中心。我们给这个结构体定义了几个方法，分别是注册用户、注销用户和获取用户在线状态。


**客户端连接处理层（client.go）：**
```go
package chat

import (
	"encoding/json"
	"server/global"
	"server/internal/message"

	"time"

	"github.com/gorilla/websocket"
	"go.uber.org/zap"
)

// 全局常量
const (
	// 服务端主动发送Ping间隔
	pingInterval = 5*time.Second 

	// 读写超时时间：超过该时间无任何响应则断开
	readWriteTimeout = 14 * time.Second 
)

// 客户端连接
type Client struct {
	UserID    int64
	Conn      *websocket.Conn
	Send      chan []byte
	Heartbeat time.Time
}

// 读消息：客户端 ——> 服务端
func (c *Client) Read() {
	defer func() {
		_ = c.Conn.Close()
		H.UnRegister(c.UserID)
		zap.S().Infof("客户端关闭: %d", c.UserID)
	}()
	
	// 客户端回复Pong帧时自动触发，刷新超时时间
	c.Conn.SetPongHandler(func(string) error {
		_ = c.Conn.SetReadDeadline(time.Now().Add(readWriteTimeout))
		_ = c.Conn.SetWriteDeadline(time.Now().Add(readWriteTimeout))
		return nil 
	})


	// 设置读取配置
	c.Conn.SetReadLimit(1024 * 4)
	c.Conn.SetReadDeadline(time.Now().Add(30 * time.Second))

	for {
		_, msgBytes, err := c.Conn.ReadMessage()
		if err != nil {
			zap.S().Errorf("读取消息错误 user:%d, err:%v", c.UserID, err)
			break
		}

		// 刷新心跳超时时间
		c.Conn.SetReadDeadline(time.Now().Add(30 * time.Second))

		// 解析消息
		var msg Message
		if err := json.Unmarshal(msgBytes, &msg); err != nil {
			zap.S().Errorf("消息解析失败: %v", err)
			continue
		}

		// 处理单聊消息
		c.handlePrivateMessage(&msg)
	}
}

// 处理单聊消息
func (c *Client) handlePrivateMessage(msg *Message) {
	// 1. 保存消息到数据库
	msgRecord := &message.Message{
		Type:     1, // 1单聊
		FromUid:  c.UserID,
		ToUid:    msg.ToUserId,
		Content:  msg.Content,
		MsgType:  msg.MsgType,
		IsRevoke: 0,
	}
	global.DB.Create(&msgRecord)

	// 2. 判断对方是否在线
	toClient, isOnline := H.GetClient(msg.ToUserId)
	if !isOnline {
		zap.S().Infof("用户 %d 不在线，消息已存库", msg.ToUserId)
		return
	}

	// 3. 转发给目标用户
	jsonData, _ := json.Marshal(map[string]interface{}{
		"from_user_id": c.UserID,
		"content":      msg.Content,
		"msg_type":     msg.MsgType,
		"time":         time.Now().Format("15:04:05"),
	})

	toClient.Send <- jsonData
	zap.S().Infof("消息转发成功: %d → %d", c.UserID, msg.ToUserId)
}

// 写消息：服务端 ——> 客户端
func (c *Client) Write() {
	defer func() {
		_ = c.Conn.Close()
	}()

	ticker := time.NewTicker(pingInterval)
	defer ticker.Stop()

	for {
		select {
		case message, ok := <-c.Send:
			if !ok {
				// 通道关闭，关闭连接
				if err := c.Conn.WriteMessage(websocket.CloseMessage, []byte{}); err != nil {
					zap.S().Errorf("通道关闭，关闭连接失败：%v", err)
					return 
				}
			}
			_ = c.Conn.SetWriteDeadline(time.Now().Add(readWriteTimeout))
			if err := c.Conn.WriteMessage(websocket.TextMessage, message); err != nil {
				zap.S().Errorf("写入客户端消息失败：%v", err)
				return 
			}

		case <-ticker.C:
			_ = c.Conn.SetWriteDeadline(time.Now().Add(readWriteTimeout))
			if err := c.Conn.WriteMessage(websocket.PingMessage, nil); err != nil {
				zap.S().Errorf("用户[%d] 发送Ping心跳失败: %v", c.UserID, err)
				return
			}
		}

	}
}
```
这里是对客户端Read和Write方法的详细实现过程。Read()是服务端监听当前客户端连接发来的消息，当从连接中读到消息时，刷新心跳超时时间，然后解析消息，这里目前只实现了单聊功能，所以直接交给了单聊处理函数，单聊处理函数中，将消息保存入数据库中，从消息结构体中，可以获得消息的目标用户的ID，然后判断目标用户是否在线，如果在线的话，就将这个消息结构体序列化写入目标用户的Send管道中。 Write()函数是服务端给客户端发送消息的实现，我定义了一个每5秒触发一次的ticker，在for select组合中有负责处理当前连接的集中情况，当当前用户的Send管道中有数据时，说明有用户给他发送消息，就执行它下边的代码，将消息取出并发送给目标用户，如果管道关闭的话，就关闭这个连接；当Send管道中没有数据时，说明没有用户给他发送消息，就执行下边的情况，就是每5秒执行一次，服务端给客户端发送Ping帧以保持连接活跃。
