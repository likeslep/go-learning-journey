# gRPC远程进程调用

## net/rpc库实现rpc方法
**rpc服务端实现**
```go
package main

import (
	"errors"
	"fmt"
	"log"
	"net"
	"net/http"
	"net/rpc"
	"os"
)

// 算数运算结构体
type Arith struct{}

type ArithRequest struct {
	A int
	B int
}

type ArithResponse struct {
	Pro int
	Quo int
	Rem int
}

func (this *Arith) Multiply(req ArithRequest, res *ArithResponse) error {
	res.Pro = req.A * req.B
	return nil
}

func (this *Arith) Divide(req ArithRequest, res *ArithResponse) error {
	if req.B == 0 {
		return errors.New("divide by zero")
	}
	res.Quo = req.A / req.B
	res.Rem = req.A % req.B
	return nil
}

func main() {
	// 注册rpc服务
	rpc.Register(new(Arith))
	// 采用http协议作为rpc载体
	rpc.HandleHTTP()

	listener, err := net.Listen("tcp", "127.0.0.1:8081")
	if err != nil {
		log.Fatalln("net.Listen() err: ", err)
	}
	defer listener.Close()

	fmt.Fprintf(os.Stdout, "%s", "start connection")

	http.Serve(listener, nil)
}
```
**rpc客户端实现**
```go
package main

import (
	"fmt"
	"log"
	"net/rpc"
)

type ArithRequest struct {
	A int
	B int
}

type ArithResponse struct {
	Pro int
	Quo int
	Rem int
}

func main() {
	conn, err := rpc.DialHTTP("tcp", "127.0.0.1:8081")
	if err != nil {
		log.Fatalln("rpc.DialHTTP() err: ", err)
	}
	defer conn.Close()

	req := ArithRequest{9, 2}
	var res ArithResponse

	err = conn.Call("Arith.Multiply", req, &res)
	if err != nil {
		log.Fatalln("conn.Call() err: ", err)
	}
	fmt.Printf("%d * %d = %d\n", req.A, req.B, res.Pro)

	err = conn.Call("Arith.Divide", req, &res)
	if err != nil {
		log.Fatalln("conn.Call() err: ", err)
	}
	fmt.Printf("%d / %d, quo is %d, rem is %d\n", req.A, req.B, res.Quo, res.Rem)
}
```

## jsonrpc库实现rpc
`net/rpc`可以实现rpc的过程，但是没办法在其它语言中调用上面例子实现的RPC方法。`net/rpc/jsonrpc`库实现的rpc方法支持跨语言调用。

**Server**

```go
package main

import (
	"errors"
	"fmt"
	"log"
	"net"
	"net/rpc"
	"net/rpc/jsonrpc"
	"os"
)

// 算数运算结构体
type Arith struct {
}

// 算数运算请求结构体
type ArithRequest struct {
	A int
	B int
}

// 算数运算响应结构体
type ArithResponse struct {
	Pro int // 乘积
	Quo int // 商
	Rem int // 余数
}

// 乘法运算方法
func (this *Arith) Multiply(req ArithRequest, res *ArithResponse) error {
	res.Pro = req.A * req.B
	return nil
}

// 除法运算方法
func (this *Arith) Divide(req ArithRequest, res *ArithResponse) error {
	if req.B == 0 {
		return errors.New("divide by zero")
	}
	res.Quo = req.A / req.B
	res.Rem = req.A % req.B
	return nil
}

func main() {
	rpc.Register(new(Arith)) // 注册rpc服务

	lis, err := net.Listen("tcp", "127.0.0.1:8090")
	if err != nil {
		log.Fatalln("fatal error: ", err)
	}
	defer lis.Close() 

	fmt.Fprintf(os.Stdout, "%s", "start connection")

	for {
		conn, err := lis.Accept() // 接收客户端连接请求
		if err != nil {
			continue
		}

		go func(conn net.Conn) { // 并发处理客户端请求
			fmt.Fprintf(os.Stdout, "%s", "new client in coming\\n")
			jsonrpc.ServeConn(conn)
		}(conn)
	}
}

```

**Client**

## gRPC和protobuf的关系
protobuf是**接口定义语言和数据序列化协议**，gRPC是基于Protobuf的RPC框架。protobuf的工作是定义数据结构、服务接口和序列化数据；RPC实现远程方法调用、通信模式和流量控制。

总结：
> Protobuf 提供“数据格式”和“服务契约”
> gRPC提供“传输机制”和“远程调用能力”
> 两者配合：Protobuf定义接口和数据，gRPC实现网络传输和调用
> gRPC的设计目标之一就是充分利用Protobuf的高效性和语言中立特性
