# 命令行工具
## 1.待办事项CLI工具
### 概述
**项目名称：**GoDo-Go语言待办事项CLI工具。
**项目目标：**开发一个功能完整、易于使用的命令行待办事项管理工具。
**目标用户：**开发人员、命令行爱好者、需要高效任务管理的用户。
**技术栈：**Go语言、JSON文件存储、标准库优先。

### 1.基础任务管理
#### 1.1 添加任务
```
# 基础语法
godo add "完成Go项目文档"

# 带优先级
godo add "修复紧急bug" -p high

# 带截至日期
godo add "提交月报" -d "2026-2-10"
```
**需求规格：**
- 必须参数：任务内容（字符串）
- 可选参数：
  - `-p, --priority`：优先级（low/medium/high，默认medium）
  - `-d, --due`：截至日期（格式：YYYY-MM-DD）
  - `-c, --category`：分类标签
- 功能：
  - 自动生成唯一ID（自增）
  - 记录创建时间
  - 默认状态为“pending”
  - 保存到JSON文件

#### 1.2 查看任务列表
```
#查看所有任务
godo list

#查看待办任务
godo list --status pending

#查看已完成任务
godo list --status completed

#按优先级筛选
godo list --priority high 

#组合筛选
godo list --status pending --priority high
```
需求规格：
- 显示字段：ID、状态图标、优先级、内容、截至日期
- 支持表格形式输出
- 支持多种排序方式（默认按创建时间倒序）
- 支持分页（--limit N）

#### 1.3 更新任务状态
```
# 标记为完成
godo complete 1

# 标记为进行中
godo start 2

# 重新打开任务
godo reopen 3 
```
需求规格：
- 状态流转：pending——>in-progress——>completed——>pending 
- 记录状态变更时间
- 支持批量操作

#### 1.4 删除任务
```
# 删除单个任务
godo delete 1

# 删除多个任务
godo delete 1 2 3 

# 删除所有已完成任务
godo clean 
```
需求规格：
- 删除前确认（可配置是否跳过）
- 支持软删除（标记为删除但不物理移除）
- 清理功能支持自定义保留天数

### 2.任务属性详细定义
#### 2.1 任务数据结构
```go 
type Task struct {
  ID int `json:"id"`    // 唯一标识，自增
  Content string `json:"content"`   // 任务内容
  Status string `json:"status"`   // pending/in-progress/completed
  Priority string `json:"priority"`   // low/medium/high
  CreateAt time.Time `json:"created_at"`    // 创建时间
  UpdatedAt time.Time `json:"updated_at"`   // 最后更新时间
  CompletedAt time.Time `json:"completed_at"`   // 完成时间（如果完成）
  DueDate time.Time `json:"due_date"`   // 截至日期
  Category string `json:"category"`   // 分类
  Tags []string `json:"tags"`   // 标签数组
}
```
