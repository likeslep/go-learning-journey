# Swagger 

## 简介
Swagger是一套用于描述、管理和测试RESTful API的规范和工具。可以把它理解为：代码 -> API描述文件 -> 可视化接口文档

## 目的
假如你写了一个注册接口 `POST /api/user/register`，正常情况下，前端不知道“请求地址是什么”、“请求方法是什么”、“参数有哪些”、“参数类型是什么”、“返回值是什么”，那么我们会写一个word文档去描述这些内容。但是这种文档有一个问题，就是如果你对代码进行了更改，但是没有对这个文档进行更改，那么前后端就开始扯皮。而Swagger的出现就是为了解决这种问题。

Swagger的核心思想：**代码即文档**

例如：
```go
// @Summary 用户注册
// @Description 注册一个新用户
// @Param data body RegisterReq true "注册信息"
// @Success 200 {object} Response 
// @Router /user/register [post]
func Register(c *gin.Context) {}
```
swagger可以通过这些注释，自动生成接口文档以及测试页面，开发者不再手动维护Word、Excel

## Swagger常用注释
1. 接口基本信息
   - @Summary : 接口简要说明
   - @Description : 接口详细描述
   - @Tags : 接口分组
2. 请求相关
   - @Accept : 请求数据格式 例如：`// @Accept json` 或 `// @Accept multipart/form-data`
   - @Produce : 响应数据格式 
3. 参数定义
   - @Param : 参数名 参数位置 参数类型 是否必须 描述 
     - Header参数 例如：JWT `// @Param        Authorization  header    string  true  "Authentication header"`
     - Query参数 例如：`GET /user?id=1` `// @Param id query int true "用户ID"`
     - Path参数 例如： `// @Param id path int true "用户ID"`
     - Body参数 例如： `// @Param data body LoginReq true "登录信息"`
4. 响应定义
   - @Success : 成功响应 
     - 返回结构体 `// @Success 200 {object} Response`
     - 返回数组 `// @Success 200 {array} Article`
     - 返回字符串 `// @Success 200 {string} string`
5. 错误响应
   - @Failure 
     - `// @Failure 400 {object} Response`
     - `// @Failure 401 {object} Response`
     - `// @Failure 500 {object} Response`
6. 路由定义
   - @Router 
     - `// @Router 路径 [方法]`

## Swagger在Gin中的使用
安装 -> 写注释 -> 生成文档并挂在路由

1. 安装：
   - 安装swag命令：`go install github.com/swaggo/swag/cmd/swag@latest`
   - 检查是否安装成功：`swag --version` 前提：把Go加入环境变量
   - 安装Gin Swagger依赖：`go get github.com/swaggo/files` `go get github.com/swaggo/gin-swagger`
2. 编写项目信息，在main.go底部添加项目的说明注释
3. 给接口写注释
4. 生成Swagger文件：
   - 进入项目根目录，执行 `swag init`
   - 执行后会在docs目录下生成文件
5. 在main.go中导入docs
    ```go
    import (
        "github.com/gin-gonic/gin"

        swaggerFiles "github.com/swaggo/files"
        ginSwagger "github.com/swaggo/gin-swagger"

        _ "blog/docs"
    )
    ```
6. 注册Swagger路由
    ```go
    r := gin.Default()
    r.POST("/api/user/login", api.Login)
    r.GET("/swagger/*any", ginSwagger.WrapHandler(swaggerFiles.Handler))
    r.Run(":8080")
    ```
